第08章 记忆与检索

1. 本章目标

LLM 本身通常是无状态的，每次请求都是独立计算。如果没有外部记忆，智能体会面临：

  • 上下文丢失：长对话早期信息会被遗忘。
  • 个性化不足：无法记住用户偏好和历史需求。
  • 学习能力弱：不能从过去经验中持续改进。
  • 一致性问题：多轮对话容易前后矛盾。

同时，LLM 内置知识也存在局限：

  • 训练数据有时间截止点。
  • 专业领域知识可能不足。
  • 容易产生事实幻觉。
  • 缺少可追溯来源。

因此需要：

  • Memory：保存用户、对话、经验和知识。
  • RAG：从外部知识库检索相关资料，辅助生成答案。

2. 记忆系统的认知启发

人类记忆可分为：

  • 感觉记忆：短暂保存感官输入。
  • 工作记忆：处理当前任务上下文。
  • 长期记忆：保存知识、经验和事件。
    • 程序性记忆：技能与习惯。
    • 陈述性记忆：可表达知识。
      • 语义记忆：概念和事实。
      • 情景记忆：具体经历。

HelloAgents 借鉴这种分层结构，构建多类型记忆模块。

核心设计是通过 MemoryTool 把记忆能力封装为标准工具，让 Agent 通过工具调用完成记忆操作。

4. 四类记忆

工作记忆 Working Memory

特点：

  • 保存当前会话中的临时信息。
  • 容量有限，默认约 50 条。
  • 纯内存存储，访问速度快。
  • 支持 TTL 自动清理。

适合保存当前对话上下文、临时目标、近期工具结果。

情景记忆 Episodic Memory

特点：

  • 保存具体事件和交互经历。
  • 支持时间序列和会话级检索。
  • 使用 SQLite + Qdrant 混合存储。

适合保存“某次对话发生了什么”“用户完成了某个里程碑”等事件。

语义记忆 Semantic Memory

特点：

  • 保存抽象知识、概念和规则。
  • 使用 Qdrant 做向量检索。
  • 使用 Neo4j 做知识图谱关系管理。
  • 支持实体和关系抽取。

适合保存用户偏好、长期事实、领域知识。

感知记忆 Perceptual Memory

特点：

  • 支持文本、图像、音频等多模态数据。
  • 按模态分别存储向量。
  • 支持同模态和跨模态检索。

适合处理截图、图片、音频等多模态交互记录。

5. MemoryTool 核心操作

MemoryTool 提供统一入口，常用操作包括：

add：添加记忆

用于把信息写入指定记忆类型。可指定：

  • content
  • memory_type
  • importance
  • metadata
  • 多模态文件路径

重要性分数用于后续检索、遗忘和整合。

search：搜索记忆

根据查询语句检索相关记忆，支持：

  • 指定记忆类型。
  • 限制返回数量。
  • 设置最低重要性阈值。
  • 混合检索与排序。

forget：遗忘记忆

支持三类遗忘策略：

  • 基于重要性：删除低重要性记忆。
  • 基于时间：删除过期记忆。
  • 基于容量：超限时删除低价值记忆。

consolidate：记忆整合

将重要短期记忆提升为长期记忆，例如：

  • 工作记忆 → 情景记忆。
  • 情景记忆 → 语义记忆。

这模拟了人类从短期记忆到长期记忆的固化过程。

6. RAG 系统

RAG 的核心流程是：

  1. 文档加载与解析。
  2. 文本分块。
  3. 向量化。
  4. 写入向量数据库。
  5. 用户提问时检索相关片段。
  6. 将检索结果注入 Prompt。
  7. LLM 基于上下文生成答案。

7. RAGTool 的价值

RAGTool 把外部知识库封装为工具，Agent 可以通过工具调用完成：

  • 文档导入。
  • 语义搜索。
  • 基于知识库问答。
  • 自动引用相关知识。

它与 MemoryTool 的区别是：

  • Memory 更偏“用户交互和智能体经验”。
  • RAG 更偏“外部文档和知识库”。

二者结合后，Agent 同时拥有个人化记忆和外部知识检索能力。

8. 本章重点

  • Memory 解决智能体的长期状态问题。
  • RAG 解决 LLM 知识过时和事实性不足的问题。
  • HelloAgents 将 Memory 和 RAG 都封装为 Tool，保持框架一致性。
  • 四类记忆分别对应不同生命周期和信息类型。
  • 记忆系统不仅要能存，还要能检索、整合和遗忘。
  • RAG 的关键是文档处理、向量化、检索和上下文构建。

9. 小结

第八章让 HelloAgents 从“无状态对话系统”升级为“有记忆、有知识检索能力的智能体系统”。记忆系统负责保存经验，RAG 系统负责接入外部知识，两者共同提升智能体在长对话、个性化任务和知识密集型任务中的表现。
