第14章 自动化深度研究智能体

1. 本章目标

深度研究助手要解决传统研究流程中的问题：

  • 搜索结果过多，信息过载。
  • 内容碎片化，缺少结构。
  • 搜索、阅读、总结、整理重复劳动多。
  • 引用来源难以追踪。

系统核心能力包括：

  1. 问题剖析：把开放主题拆成可检索子任务。
  2. 多轮信息采集：调用不同搜索引擎持续收集资料。
  3. 反思与总结：识别阶段信息缺口，整合成结构化报告。

2. 技术架构

项目仍采用前后端分离架构：

数据流：

  1. 用户输入研究主题。
  2. 前端通过 SSE 连接后端。
  3. 后端创建研究状态。
  4. PlannerAgent 拆分 TODO。
  5. 对每个 TODO 执行搜索和总结。
  6. NoteTool 保存中间结果。
  7. ReportWriter 生成最终报告。
  8. 后端流式推送进度和结果。

3. TODO 驱动研究范式

TODO 驱动研究把“研究一个主题”拆为“规划 → 执行 → 整合”。

例如研究“Datawhale 是什么组织”：

每个 TODO 都包含：

  • title：任务标题。
  • intent：研究意图。
  • query：搜索查询语句。

这种方式让研究过程结构化、可追踪，也便于阶段性保存和复盘。

4. 三阶段研究流程

规划阶段

TODO Planner 将研究主题拆成 3-5 个子任务。一个好的规划应满足：

  • 覆盖全面。
  • 逻辑清晰。
  • 查询精准。
  • 数量适中。
  • 适合搜索引擎检索。

Planner 的输出通常是 JSON，便于后端解析为 TodoItem。

执行阶段

每个子任务依次执行：

  1. 使用 SearchTool 搜索资料。
  2. 提取标题、URL、摘要。
  3. 调用 Task Summarizer 总结。
  4. 使用 NoteTool 保存结果。
  5. 向前端推送进度。

总结内容需要包含核心观点、关键数据和来源引用。

报告阶段

Report Writer 整合所有子任务总结，生成最终 Markdown 报告。报告通常包含：

  • 标题。
  • 概述。
  • 各子任务详细分析。
  • 总结。
  • 参考文献。

5. 三个核心 Agent

TODO Planner

职责：将研究主题拆分为子任务。重点是理解主题、设计合理研究路径和生成搜索查询。

Task Summarizer

职责：阅读搜索结果，提取关键信息，合并相似观点，并保留来源引用。

Report Writer

职责：把多个子任务总结整合为结构化报告，消除重复信息，统一格式和引用。

三个 Agent 是顺序协作关系：Planner 的输出决定执行任务，Summarizer 生成中间结论，Report Writer 负责最终整合。

6. ToolAwareSimpleAgent

本章扩展了 SimpleAgent，加入工具调用监听能力，形成 ToolAwareSimpleAgent。

它的价值包括：

  • 记录 Agent 调用了哪些工具。
  • 捕获工具参数和结果。
  • 支持调试与日志分析。
  • 向前端实时展示研究过程。

实现方式是重写工具执行方法，在每次工具调用后触发 tool_call_listener 回调。

7. SearchTool 扩展

深度研究依赖高质量搜索。本章扩展 SearchTool，支持：

  • Tavily。
  • DuckDuckGo。
  • Perplexity。
  • SearXNG。
  • Advanced 混合搜索模式。

搜索结果需要做后处理：

  • 去重 URL。
  • 限制单个来源 token 数。
  • 保留标题、链接和摘要。
  • 记录搜索后端和错误提示。

8. NoteTool 的作用

NoteTool 用于持久化研究过程。每个子任务可保存为 Markdown 笔记，最终报告也可保存到 workspace。

典型结构：

这让研究过程可追溯、可恢复，也便于后续审查来源和中间结论。

9. SSE 流式体验

研究任务耗时较长，前端不能只等待最终结果。本章使用 SSE 实时推送：

  • 当前状态。
  • 任务列表。
  • 搜索进度。
  • 工具调用日志。
  • 子任务完成事件。
  • 最终报告。

这提升了用户体验，也让智能体过程更透明。

10. 本章重点

  • 深度研究适合 TODO 驱动的任务拆解。
  • 多 Agent 分别负责规划、总结和报告。
  • SearchTool 决定资料质量，NoteTool 保证过程可追踪。
  • ToolAwareSimpleAgent 增强了可观测性。
  • SSE 让长任务具备实时反馈。

11. 小结

第十四章展示了知识密集型智能体应用的构建方式。深度研究助手的关键不是单次回答，而是规划、搜索、总结、记录和报告的完整闭环。
