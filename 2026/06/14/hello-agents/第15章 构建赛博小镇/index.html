第15章 构建赛博小镇

1. 本章目标

传统游戏 NPC 多依赖固定台词或预设对话树，缺少真实互动。本章的赛博小镇希望让 NPC 具备更强的“生命感”：

  • 能理解玩家自然语言。
  • 能根据角色设定回复。
  • 能记住历史互动。
  • 能根据好感度改变态度。
  • 能在 2D 场景中与玩家实时交互。

核心功能包括：

  • 智能 NPC 对话系统。
  • 短期记忆与长期记忆。
  • 好感度系统。
  • Godot 2D 像素风办公室场景。
  • 对话与状态日志。

2. 技术架构

项目采用“游戏引擎 + 后端服务”分离架构：

数据流：

  1. 玩家在 Godot 中靠近 NPC 并按 E 交互。
  2. Godot 通过 HTTP 请求发送对话到 FastAPI。
  3. 后端调用对应 NPC Agent。
  4. Agent 检索记忆并调用 LLM 生成回复。
  5. 后端更新好感度和状态，写入日志。
  6. Godot 显示 NPC 回复。

3. NPC Agent 系统

每个 NPC 都是一个独立的 SimpleAgent，拥有自己的角色设定、系统提示词、记忆和状态。

示例 NPC：

  • 张三：Python 工程师，严谨专业，喜欢分享技术知识。
  • 李四：产品经理，外向善沟通，重视用户体验。
  • 王五：UI 设计师，温和有创意，关注视觉表达。

通过不同系统提示词，LLM 会以不同说话风格和角色身份回应玩家。

4. 记忆系统

NPC 需要记住与玩家的互动，才能形成连续关系。项目使用两类记忆：

短期记忆

保存最近对话，容量有限，用于保持当前上下文连贯。适合解决“它”“刚才说的”等指代问题。

长期记忆

保存历史互动，使用数据库和向量检索。玩家提到过去事件时，NPC 可以检索相关记忆并回忆。

对话处理流程：

  1. 获取最近短期记忆。
  2. 根据玩家输入检索长期记忆。
  3. 构建上下文。
  4. 调用 Agent 生成回复。
  5. 将本轮互动写入记忆。

5. 批量对话生成

如果所有 NPC 的背景行为都实时调用 LLM，成本和延迟会很高。因此本章设计了轻负载的批量生成机制。

批量生成的思路是：一次 LLM 调用为多个 NPC 生成背景对话或状态描述，并以 JSON 返回。

适合场景：

  • NPC 背景自言自语。
  • 定时更新场景氛围。
  • 根据时间生成办公室状态。
  • 降低 API 调用成本。

玩家主动对话时仍使用即时响应，确保个性化和准确性。

6. 混合模式

项目采用“批量生成 + 即时响应”的混合模式：

  • 背景对话使用批量生成，低成本、低延迟。
  • 玩家交互使用专属 Agent，即时生成个性化回复。

这种设计既让 NPC 平时看起来“活着”，又保证玩家对话质量。

7. 好感度系统

好感度让 NPC 与玩家关系逐步变化。项目设计五个等级：

  • 陌生：礼貌但疏远。
  • 熟悉：正常交流。
  • 友好：愿意分享更多信息。
  • 亲密：信任玩家，回复更热情。
  • 挚友：亲切真诚，无话不谈。

好感度根据玩家消息和 NPC 回复进行情感分析后调整。例如友好表达加分，普通交流小幅加分，不友好表达扣分。

8. 好感度如何影响对话

好感度不是单纯数值，而会进入 NPC 的系统提示词，影响回复风格：

  • 分数低：回复简短、保持距离。
  • 分数中等：自然友好。
  • 分数高：更热情、更详细、更愿意分享。

这让玩家能感受到关系变化，增强沉浸感。

9. 后端服务设计

FastAPI 后端包含：

  • NPCAgentManager：管理 NPC Agent。
  • RelationshipManager：管理好感度。
  • StateManager：管理 NPC 忙碌状态、位置和动作。
  • DialogueLogger：记录对话和错误日志。

核心 API 包括：

  • 获取所有 NPC 状态。
  • 获取单个 NPC 状态。
  • 玩家与 NPC 对话。
  • 查询玩家与 NPC 的好感度。

对话接口需要处理并发，避免同一个 NPC 同时与多个玩家对话。

10. Godot 场景系统

Godot 的核心是节点和场景。项目包含四个核心场景：

  • Main：主场景，组合背景、玩家、NPC 和 UI。
  • Player：玩家角色，负责移动、碰撞、摄像机。
  • NPC：通用 NPC 模板，通过参数实例化张三、李四、王五。
  • DialogueUI：对话界面。

使用场景实例化可以复用 NPC 模板，减少重复开发。

11. 本章重点

  • AI NPC 的关键是角色设定、记忆和关系变化。
  • Godot 负责交互体验，FastAPI 负责 AI 后端。
  • 批量生成适合背景内容，即时响应适合玩家对话。
  • 好感度系统让 NPC 行为具备成长性。
  • 日志系统对调试和观察 NPC 行为很重要。

12. 小结

第十五章展示了智能体在游戏中的应用方式。赛博小镇不仅是一个 AI NPC Demo，也展示了智能体系统如何与实时交互场景结合：角色、记忆、状态、关系和前端体验共同构成“有生命感”的智能体应用。
