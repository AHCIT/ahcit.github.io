大语言模型基础

1. 语言模型的演进

语言模型（LM） 的核心任务是计算词序列出现的概率，是智能体理解指令、生成回答的基础。

1.1 N-gram 模型

N-gram 基于马尔可夫假设：当前词只依赖前面有限的 n-1 个词。

模型 假设
Bigram 当前词只依赖前 1 个词
Trigram 当前词只依赖前 2 个词

Bigram 概率估计：

$$P(w_i|w_{i-1})=\frac{Count(w_{i-1},w_i)}{Count(w_{i-1})}$$

局限：

  • 数据稀疏：未出现过的词序列概率为 0
  • 泛化差：无法理解词语语义相似性

1.2 词嵌入与神经网络语言模型

神经网络语言模型用连续向量表示词，即词嵌入（Word Embedding）。

核心思想：

  • 将词映射到高维语义空间
  • 语义相近的词向量距离更近
  • 用神经网络学习上下文到下一个词的映射

典型语义关系：

词嵌入解决了 N-gram 泛化差的问题，但仍受固定上下文窗口限制。

1.3 RNN 与 LSTM

RNN 通过隐藏状态引入“记忆”，可处理序列信息：

问题：长序列训练时容易出现梯度消失/爆炸，难以捕捉长期依赖。

LSTM 通过细胞状态和门控机制缓解长期依赖：

门 作用
遗忘门 决定丢弃旧信息
输入门 决定写入新信息
输出门 决定输出哪些信息

2. Transformer 架构

Transformer 于 2017 年提出，抛弃 RNN 的循环结构，依靠注意力机制捕捉序列依赖，实现并行计算。

2.1 Encoder-Decoder

组件 作用
Encoder 理解输入序列，生成上下文表示
Decoder 根据已生成内容和 Encoder 输出生成目标序列

2.2 自注意力机制

Self-Attention 让模型处理每个词时都能关注序列中的其他词，并分配不同权重。

每个词会生成三个向量：

向量 含义
Query（Q） 当前词发出的查询
Key（K） 被查询词的索引
Value（V） 被查询词携带的信息

注意力公式：

$$Attention(Q,K,V)=softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

计算过程：生成 Q/K/V → 计算相关性 → 缩放 → Softmax → 对 V 加权求和。

2.3 多头注意力

Multi-Head Attention 将 Q/K/V 切分为多个头，让不同头从不同角度捕捉关系，最后拼接整合。

优势：

  • 同时关注不同位置
  • 捕捉不同语义关系
  • 增强表达能力

2.4 前馈网络、残差与归一化

每层 Transformer 包含逐位置前馈网络（FFN）：

$$FFN(x)=max(0,xW_1+b_1)W_2+b_2$$

同时使用 Add & Norm：

机制 作用
残差连接 缓解梯度消失，支持深层训练
层归一化 稳定输入分布，加速收敛

2.5 位置编码

自注意力本身不包含顺序信息，因此需要位置编码（Positional Encoding），给词向量加入位置信息，使模型区分词序。

3. Decoder-Only 架构

GPT 采用 Decoder-Only：抛弃 Encoder，只保留 Decoder。

核心思想：

语言生成本质上就是不断预测下一个最可能出现的词。

3.1 自回归生成

3.2 掩码自注意力

为了防止模型“偷看未来”，Decoder-Only 使用 Masked Self-Attention：将当前位置之后的注意力分数置为极小值，使其 Softmax 权重接近 0。

3.3 优势

  • 训练目标统一：预测下一个词
  • 结构简单，易扩展到超大规模
  • 天然适合对话、写作、代码生成等生成任务

4. 与大语言模型交互

4.1 提示工程

提示工程（Prompt Engineering） 是设计输入提示，引导模型产生期望输出的方法。

采样参数

参数 作用 特点
Temperature 控制随机性 越低越确定，越高越发散
Top-k 只保留概率最高的 k 个候选 k=1 退化为贪心采样
Top-p 保留累计概率达到 p 的候选集 动态适应分布

建议：

场景 Temperature
事实问答、代码、计算 0 - 0.3
日常对话、常规写作 0.3 - 0.7
创意写作、头脑风暴 0.7 - 2

同时设置时通常顺序为：Temperature → Top-k → Top-p。

Prompt 类型

类型 说明
Zero-shot 不给示例，直接下达任务
One-shot 给 1 个示例说明格式
Few-shot 给多个示例帮助模型理解边界

指令调优

Instruction Tuning 使用大量“指令-回答”数据继续训练模型，使模型从“文本补全”变为“理解并执行指令”。

常用技巧

  • 角色扮演：限定身份、风格和知识范围
  • 上下文示例：用示例规定输出格式
  • 思维链（CoT）：引导模型逐步推理，提升复杂任务准确率

5. 文本分词

Tokenization 是将文本转换为模型可处理 Token 序列的过程。

5.1 分词方式

方法 优点 缺点
按词分词 语义完整 词表爆炸，存在 OOV
按字符分词 词表小，无 OOV 语义弱，学习效率低
子词分词 平衡词表和语义 现代 LLM 主流方案

5.2 BPE 算法

BPE（Byte-Pair Encoding） 是主流子词算法之一。

流程：

  1. 词表初始化为基本字符
  2. 统计相邻 Token 对频率
  3. 合并最高频 Token 对
  4. 重复直到词表达到目标大小

常见算法：

算法 特点 代表模型
BPE 合并最高频相邻 Token 对 GPT
WordPiece 合并能最大化语料概率的 Token 对 BERT
SentencePiece 空格也作为字符，语言无关、可逆 Llama

5.3 对开发者的意义

  • 上下文窗口按 Token 计算，不是字符数
  • API 通常按 Token 计费
  • 空格、大小写、符号会影响分词结果，进而影响模型表现

7. 模型选择

模型选型是在性能、成本、速度、部署方式之间权衡，而不是单纯追求最大最强。

7.1 关键因素

维度 说明
性能与能力 推理、代码、创作、多语言等
成本 API Token 成本或本地硬件成本
速度 实时场景关注延迟
上下文窗口 长文档、代码库、长期记忆需要大窗口
部署方式 API 简单，本地部署更隐私可控
生态工具链 社区、框架、微调工具是否成熟
可微调性 是否支持领域定制
安全伦理 偏见、毒性、幻觉、合规性

7.2 闭源 vs 开源

类型 优势 劣势 代表
闭源 开箱即用、性能强、服务稳定 成本高、可控性弱、数据需发给第三方 GPT、Gemini、Claude
开源 灵活、透明、本地部署、可微调 需要硬件与运维 Llama、Mistral、Qwen、ChatGLM

8. 缩放法则与局限性

8.1 缩放法则

Scaling Laws 揭示模型性能与参数量、训练数据量、计算资源之间存在可预测的幂律关系。

Chinchilla 定律指出：给定计算预算下，模型参数量和训练数据量存在最优配比，模型不是越大越好，数据量同样关键。

8.2 涌现能力

当模型规模达到一定阈值后，会出现小模型不具备的新能力：

  • 指令遵循
  • 思维链推理
  • 多步推理
  • 代码生成
  • 上下文学习

8.3 模型幻觉

幻觉（Hallucination） 指模型生成与事实、输入或上下文不一致的内容。

类型 说明
事实性幻觉 与现实事实不符
忠实性幻觉 摘要/翻译未忠实反映源文本
内在幻觉 与输入信息直接矛盾

成因：

  • 训练数据错误或矛盾
  • 自回归机制只预测下一个 Token，没有事实核查
  • 复杂推理链出错
  • 知识时效性不足
  • 训练数据包含偏见

缓解方法：

层面 方法
数据层面 数据清洗、事实知识、RLHF
模型层面 表达不确定性、改进架构
推理层面 RAG、多步验证、外部工具调用

RAG（检索增强生成） 是缓解幻觉的重要方法：先检索外部知识，再让模型基于事实上下文生成回答。

9. 本章要点总结

  1. 语言模型的目标是预测词序列概率
  2. N-gram 简单但有数据稀疏和泛化差的问题
  3. 词嵌入让模型获得语义泛化能力
  4. RNN/LSTM 引入序列记忆，但并行效率不足
  5. Transformer 用注意力机制实现长距离依赖建模和并行计算
  6. Decoder-Only 架构通过预测下一个词支撑现代生成式大模型
  7. 提示工程、采样参数和分词都会直接影响模型输出
  8. 模型选型需综合性能、成本、速度、上下文、部署和安全
  9. 缩放法则推动 LLM 能力提升，但幻觉、偏见和知识过时仍需工程化缓解
