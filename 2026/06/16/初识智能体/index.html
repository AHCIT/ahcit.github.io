初识智能体

1. 智能体定义

智能体（Agent） 是能够通过传感器（Sensors） 感知环境（Environment），并自主地通过执行器（Actuators） 采取行动（Action） 以达成特定目标的实体。

四个基本要素：

要素 说明 示例
环境 智能体所处的外部世界 道路交通、金融市场、游戏世界
传感器 感知环境状态的媒介 摄像头、麦克风、API 数据流
执行器 对环境施加影响的设备 机械臂、方向盘、代码执行、API 调用
自主性 独立决策以达成目标的能力 智能体的"灵魂"

核心闭环：感知 → 决策 → 行动 → 观察，循环往复。

2. 传统智能体的演进

2.1 简单反射智能体（Simple Reflex Agent）

  • 基于"条件-动作"规则，无记忆、无预测能力
  • 仅依赖当前感知输入，如自动恒温器
  • 优点：可靠高效；缺点：无法处理需要上下文的复杂任务

2.2 基于模型的反射智能体（Model-Based Reflex Agent）

  • 引入内部世界模型（World Model），追踪无法直接感知的环境状态
  • 拥有初级"记忆"，决策基于更完整的世界状态理解
  • 如：隧道中行驶的自动驾驶汽车

2.3 基于目标的智能体（Goal-Based Agent）

  • 主动、有预见性地选择能导向特定目标的行动
  • 核心能力：规划（Planning），如 GPS 导航中的 A* 算法

2.4 基于效用的智能体（Utility-Based Agent）

  • 为每个世界状态赋予效用值，目标是最大化期望效用
  • 能在冲突目标间权衡，如：时间最短 vs 路程最省油 vs 避开拥堵

2.5 学习型智能体（Learning Agent）

  • 包含性能元件（执行决策）和学习元件（通过经验改进策略）
  • 代表路径：强化学习（RL），如 AlphaGo Zero 通过自我对弈发现超越人类的棋路

3. LLM 驱动的新范式

3.1 与传统智能体的核心对比

维度 传统智能体 LLM 智能体
核心引擎 显式规则/模型 预训练大语言模型
知识来源 工程师显式编程 海量数据隐式学习
行为模式 确定、有边界 灵活、通用、涌现
交互方式 结构化输入输出 自然语言

3.2 LLM 智能体的关键能力

  • 规划与推理：将高层级目标分解为逻辑子任务
  • 工具使用：主动调用外部 API 补全信息缺口
  • 动态修正：根据反馈实时调整行为策略

4. 智能体的三种分类维度

4.1 基于内部决策架构

反应式 → 模型式 → 基于目标 → 基于效用（学习能力为可赋予所有类型的元能力）

4.2 基于时间与反应性

类型 特点 优势 劣势
反应式 近乎即时响应，无/极少规划 速度快、计算开销低 短视，易陷入局部最优
规划式 行动前系统探索、评估后果 战略性、有远见 时间与计算成本高
混合式 结合反应与规划 兼顾即时性与长远性 架构复杂

LLM 智能体在"思考-行动-观察"循环中实现了灵活的混合模式。

4.3 基于知识表示

范式 核心思想 优势 劣势
符号主义 AI 符号的逻辑操作，显式规则 透明、可解释 脆弱，知识获取瓶颈
亚符号主义 AI 神经网络从数据中学习统计模式 模式识别强、鲁棒 黑箱、不可解释、可能幻觉
神经符号主义 AI 融合两者优点 学习 + 推理 仍在发展中

类比卡尼曼双系统理论：系统 1（快速直觉）= 亚符号主义，系统 2（慢速推理）= 符号主义。

5. 智能体的构成与运行原理

5.1 PEAS 模型

描述任务环境的框架：

  • Performance（性能度量）：如何评价成功
  • Environment（环境）：所处的外部世界
  • Actuators（执行器）：可执行的动作
  • Sensors（传感器）：感知输入

5.2 环境特性

  • 部分可观察：无法获取全部信息，需记忆与探索
  • 随机性：行动结果不确定，需处理不确定性
  • 多智能体：存在其他行动者相互影响
  • 序贯性：当前动作影响未来
  • 动态性：环境在决策时持续变化

5.3 智能体循环（Agent Loop）

思考阶段包含两个子环节：

  1. 规划（Planning）：更新理解，制定/调整计划，分解子任务
  2. 工具选择（Tool Selection）：从工具库中选择合适工具并确定参数

5.4 交互协议：Thought-Action-Observation

  • Thought：内部推理的"快照"，阐述分析、反思与计划
  • Action：对环境的具体操作，以函数调用形式表示
  • Observation：环境返回的结果，经处理后作为下一轮输入

6. 动手实践：构建智能旅行助手

6.1 核心组件

  • 指令模板（System Prompt）：定义角色、可用工具、输出格式
  • 工具函数：get_weather(city) 调用 wttr.in API，get_attraction(city, weather) 调用 Tavily Search API
  • LLM 客户端：兼容 OpenAI 接口规范的通用客户端
  • 主循环：构建 Prompt → 调用 LLM → 解析 Action → 执行工具 → 记录 Observation → 循环

6.2 执行流程示例

四项基本能力：任务分解、工具调用、上下文理解、结果合成。

7. 智能体应用的协作模式

7.1 作为开发者工具

深度集成到开发工作流，增强而非取代开发者：

  • GitHub Copilot：代码自动补全 + 对话式编程
  • Claude Code：终端 AI 编程助手，支持全流程开发
  • Trae：轻量级代码生成与优化
  • Cursor：AI 原生代码编辑器，理解整个代码库上下文

7.2 作为自主协作者

从"命令-执行"到"目标-委托"的范式转变：

架构范式 代表框架 特点
单智能体自主循环 AgentGPT 思考-规划-执行-反思闭环
多智能体协作 CAMEL, MetaGPT, CrewAI, AutoGen 角色扮演/组织化工作流
高级控制流 LangGraph 状态图建模，支持循环、分支、回溯

7.3 Workflow vs Agent

维度 Workflow Agent
本质 预定义的静态流程编排 自主的目标导向系统
决策 固定规则（if-else） 动态推理与决策
适用 流程明确、规则清晰 目标模糊、需灵活应对
示例 费用报销审批流程 智能旅行助手

核心区别：Workflow 让 AI 按部就班执行指令，Agent 赋予 AI 自由度自主达成目标。

8. 本章要点总结

  1. 智能体 = 传感器 + 执行器 + 自主性 + 环境交互
  2. 传统智能体从简单反射到学习型，能力逐步增强
  3. LLM 智能体通过预训练获得涌现能力，以自然语言交互
  4. 三种分类维度：决策架构、时间反应性、知识表示
  5. PEAS 模型描述任务环境，Agent Loop 是核心运行机制
  6. Thought-Action-Observation 是 LLM 智能体的标准交互协议
  7. 协作模式分为"开发者工具"和"自主协作者"两类
  8. Workflow 是固定流程，Agent 是自主决策系统
