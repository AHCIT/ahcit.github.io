第10章 智能体通信协议

1. 本章目标

在没有协议时，每接入一个外部服务都要手写 Tool：GitHub、数据库、文件系统、天气 API 等都需要独立适配。这会带来：

  • 重复代码多。
  • 难以维护。
  • 工具不可复用。
  • 无法动态发现能力。
  • 多智能体协作困难。

通信协议的价值是提供统一接口，让 Agent 以标准方式访问外部能力或其他 Agent。

2. 三种协议定位

MCP：Agent 与工具通信

MCP（Model Context Protocol）用于标准化智能体与外部工具、资源、提示模板之间的通信。

可以把 MCP 理解为智能体工具生态的“USB-C”：不同工具只要遵循 MCP，就能被兼容客户端调用。

适合场景：

  • 文件系统访问。
  • 数据库查询。
  • GitHub、Slack 等 API 集成。
  • 本地或远程工具服务。

A2A：Agent 与 Agent 通信

A2A（Agent-to-Agent Protocol）用于智能体之间的点对点协作。它关注任务委托、状态同步和工件传递。

适合场景：

  • 研究员 Agent + 撰写员 Agent + 编辑 Agent。
  • 多角色协作完成复杂任务。
  • 去中心化 Agent 协同。

ANP：Agent 网络协议

ANP（Agent Network Protocol）面向大规模智能体网络，重点是服务注册、发现和路由。

适合场景：

  • 大规模 Agent 生态。
  • 动态服务发现。
  • 多 Agent 网络基础设施。

3. 如何选择协议

  • 需要访问外部工具或资源：选 MCP。
  • 需要多个 Agent 点对点协作：选 A2A。
  • 需要构建大规模 Agent 网络：考虑 ANP。

目前 MCP 生态最成熟，实际项目中优先使用大厂或社区维护活跃的 MCP Server。

5. MCP 核心概念

MCP 架构包含三类角色：

  • Host：用户交互入口，如 Claude Desktop 或 Agent 应用。
  • Client：负责连接 MCP Server 并收发协议消息。
  • Server：提供工具、资源或提示模板。

MCP 的核心能力有三种：

  1. Tools：主动执行操作，如读文件、查数据库。
  2. Resources：被动提供数据，如文件内容、文档资源。
  3. Prompts：提供可复用提示模板。

6. MCP 与 Function Calling 的关系

Function Calling 是模型能力，解决“模型如何决定调用函数、生成参数”。

MCP 是工程协议，解决“工具如何被描述、发现、连接和调用”。

二者不是竞争关系，而是互补关系。Function Calling 偏模型侧，MCP 偏基础设施侧。

7. MCPTool 自动展开

HelloAgents 的 MCPTool 支持自动展开机制：

  1. 连接 MCP Server。
  2. 发现服务端提供的工具。
  3. 为每个工具创建包装器。
  4. 注册到 Agent 的工具系统。
  5. Agent 像调用普通工具一样调用 MCP 工具。

例如一个文件系统 MCP Server 可展开为：

  • fs_read_file
  • fs_write_file
  • fs_list_directory

这样 Agent 不需要关心 MCP 的底层 JSON-RPC 调用细节。

8. MCP 传输方式

HelloAgents 支持多种 MCP 传输方式：

  • Memory：内存传输，适合测试。
  • Stdio：标准输入输出，适合本地开发和社区 MCP Server。
  • HTTP：适合远程服务和生产环境。
  • SSE：适合实时流式通信。
  • StreamableHTTP：适合双向流式 HTTP 场景。

实际使用中，MCPTool 更常用于 Memory 和 Stdio；HTTP/SSE 场景更适合直接使用底层 MCPClient。

9. A2A 协议重点

A2A 解决的是智能体协作问题。它避免所有通信都经过中心协调器，采用点对点模式，减少单点故障和性能瓶颈。

核心抽象包括：

  • Task：待完成的任务。可创建、协商、执行、完成或失败。
  • Artifact：任务执行产生的工件，如报告、代码、文件。

A2A 的关键价值是让 Agent 具备任务委托、状态跟踪和协商能力。

10. ANP 协议重点

ANP 关注大规模 Agent 网络中的服务发现。它包含：

  • 服务注册。
  • 服务发现。
  • 服务路由。
  • 网络管理。

在 HelloAgents 中，ANP 更偏概念性模拟，用于帮助理解 Agent 网络基础设施。

11. 本章重点

  • 通信协议让 Agent 能标准化连接工具、资源和其他 Agent。
  • MCP 是当前最实用、生态最成熟的工具通信协议。
  • A2A 面向多智能体点对点协作。
  • ANP 面向大规模 Agent 网络发现与路由。
  • HelloAgents 将协议封装为 Tool，保持统一调用方式。
  • MCPTool 的自动展开机制大幅降低工具接入成本。

12. 小结

第十章把 HelloAgents 从单体智能体扩展到协议化生态。通过 MCP，Agent 可以快速接入外部工具；通过 A2A，多个 Agent 可以协作；通过 ANP，可以理解更大规模的智能体网络设计。
